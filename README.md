# Personal Telegram Reminder & People Bot

Личный Telegram-бот на aiogram 3.x: напоминания с гибкими алертами и
мини-CRM по знакомым с дедупликацией имён. Свободный текст разбирается через
Anthropic API (модель `claude-haiku-4-5-20251001`) в структурированный intent.

## Возможности

- ⏰ Напоминания из свободного текста («позвонить бабушке сегодня в 12»),
  несколько вариантов «напомнить заранее», отложенные и повторяющиеся
  напоминания (каждый день / каждую неделю).
- 👤 Мини-CRM: карточки людей с алиасами, заметками, днями рождения;
  fuzzy-поиск (`rapidfuzz`) для дедупликации при упоминании имени.
- 🎂 Авто-напоминания о днях рождения — ежегодное напоминание создаётся/
  обновляется/удаляется автоматически при любом изменении даты рождения,
  время настраивается через `BIRTHDAY_ALERT_TIME`.
- 🔎 Полнотекстовый поиск по заметкам (SQLite FTS5) — «кто любит рыбалку?».
- 🔒 Бот приватный — отвечает только владельцу (`ALLOWED_USER_IDS`).
- 💾 Напоминания переживают перезапуск — задачи APScheduler персистятся в
  отдельном SQLite-файле.
- 📦 `/export` — бэкап всех людей и напоминаний в JSON-файл, присылается
  документом в чат.
- 📊 `/stats` — количество людей, заметок и активных напоминаний в базе.
- 🗂 Все файлы на диске (обе БД, логи, временные экспорты) живут строго
  внутри `DATA_DIR` — важно для деплоя на Railway с примонтированным томом.

## Стек

Python 3.11+, aiogram 3.x, SQLAlchemy 2.x (async, SQLite/aiosqlite),
APScheduler (AsyncIOScheduler + SQLAlchemyJobStore), Anthropic API, rapidfuzz.

## Установка

1. **Создать бота у BotFather**
   - Напишите [@BotFather](https://t.me/BotFather) в Telegram, отправьте
     `/newbot`, следуйте инструкциям и сохраните выданный токен.
   - Узнайте свой Telegram user ID (например, через [@userinfobot](https://t.me/userinfobot)) —
     он понадобится для `ALLOWED_USER_IDS`, чтобы бот отвечал только вам.

2. **Установить зависимости**

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```

3. **Настроить `.env`**

   ```bash
   cp .env.example .env
   ```

   Заполните файл:

   ```env
   TELEGRAM_BOT_TOKEN=токен-от-BotFather
   ANTHROPIC_API_KEY=ваш-ключ-Anthropic
   TIMEZONE=Europe/Kyiv
   ALLOWED_USER_IDS=ваш_telegram_id
   DATA_DIR=./data
   BIRTHDAY_ALERT_TIME=15:00
   ```

   `DATA_DIR` — папка для всех файлов, которые должны пережить перезапуск
   (обе БД, логи, временные экспорты для `/export`). **На Railway** (или
   любой платформе с эфемерной файловой системой) обязательно укажите тут
   путь к примонтированному volume, например `/app/data` — иначе база
   исчезнет при каждом редеплое.

4. **Запустить бота**

   ```bash
   python3 bot.py
   ```

   При первом запуске автоматически создастся `<DATA_DIR>/bot.db` (основная
   база) и `<DATA_DIR>/scheduler.db` (хранилище задач APScheduler, отдельный
   файл — так избегаются конфликты блокировок SQLite между async-движком и
   синхронным jobstore'ом). Логи пишутся в `<DATA_DIR>/bot.log` и в консоль.
   При старте в лог выводятся резолвленные абсолютные пути обеих БД —
   удобно проверить в логах Railway, что они указывают на смонтированный том.

## Структура проекта

```
bot.py                  — точка входа, инициализация бота/БД/планировщика
config.py               — конфигурация из .env
db/
  models.py             — модели SQLAlchemy (Person, Reminder, ...)
  repo.py               — доступ к БД, инициализация FTS5
handlers/
  common.py             — /start, /help, диспетчер intent → обработчик, access control
  reminders.py          — сценарии напоминаний
  people.py             — сценарии мини-CRM (люди, дедупликация, поиск)
  states.py             — FSM-состояния диалогов
services/
  llm_parser.py         — обращение к Anthropic API, разбор JSON-интентов
  conversation.py       — короткая история диалога для контекста классификатора
                          + детерминированная проверка явных слов отмены
  scheduler.py          — APScheduler: планирование и срабатывание напоминаний
  person_matcher.py     — fuzzy-поиск дублей людей (rapidfuzz)
  birthdays.py          — авто-напоминания о днях рождения (+ idempotent sync при старте)
scripts/
  verify_birthdays.py   — самопроверка: DATA_DIR, создание/обновление/удаление
                          дня рождения, идемпотентная синхронизация при старте
  eval_classifier.py    — прогон ~30 фраз через реальный Anthropic API,
                          таблица pass/fail для классификатора intent'ов
```

## Классификатор intent'ов: контекст и детерминизм

- Классификатор вызывается с `temperature=0` для воспроизводимости.
- Каждому вызову передаётся текст последних сообщений (`services/conversation.py`,
  до 6 последних реплик пользователя/бота), чтобы короткие ответы вроде
  «сегодня», «в 12», «да» правильно понимались как ответ на предыдущий
  уточняющий вопрос бота, а не классифицировались из ничего.
- Отмена/удаление (`delete_reminder`) распознаётся ДЕТЕРМИНИРОВАННО по явным
  словам («отмени», «удали», «убери», «не надо», «стоп») ещё ДО обращения к
  LLM — так фразы вроде «поздравить с днём рождения» никогда не могут быть
  ошибочно приняты за отмену, независимо от модели.
- Каждый вызов классификатора логируется одной строкой
  (`LLM_CLASSIFY | in=... | context=... | out=...`) — удобно грепать в логах,
  если бот на что-то отреагировал не так.
- `python3 scripts/eval_classifier.py` прогоняет набор тестовых фраз (включая
  все сообщённые баги) через реальный API и печатает таблицу pass/fail —
  использовать при любых правках промпта.

## Примеры сообщений

- «позвонить бабушке сегодня в 12»
- «напоминай пить воду каждый день в 9»
- «новый знакомый Валера, день рождения 12 июля, любит рыбалку»
- «у Валеры есть собака Рекс»
- «кто любит рыбалку?»
- «мои напоминания» / «мои люди»

## Заметки по реализации

- FTS5 не умеет в русскую морфологию «из коробки» (не найдёт «рыбалка» по
  заметке «любит рыбалку», так как это разные словоформы) — поиск по
  заметкам поэтому в первую очередь пробует FTS5-запрос, а если он ничего
  не находит, откатывается на `LIKE`-подстроку.
- Бот не падает от неожиданного/невалидного ответа модели: JSON парсится
  через `json.loads` с обработкой ошибок, при сбое возвращается intent
  `unknown` и бот вежливо переспрашивает.
